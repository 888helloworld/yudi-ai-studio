# 御弟哥哥 AI 图文创作工作台

一个给运营、卖家、博主和内容团队用的 AI 图文小工作台。

不用把自己逼成“提示词工程师”，也不用在一堆工具之间来回搬东西。你只管描述你想要的画面、文案、风格和用途，剩下的交给它慢慢干活。

## 先说人话：它能帮你做什么？

| 你想做的事 | 可以用它干什么 |
| --- | --- |
| 想要一张图 | 写一句描述，直接生图 |
| 手里有参考图 | 上传参考图，让 AI 按感觉改一版 |
| 不会写提示词 | 丢一张图进去，让它帮你反推提示词 |
| 做小红书内容 | 生成标题、正文、标签、配图 |
| 做商品素材 | 快速尝试不同构图、背景、光线和比例 |
| 想批量试图 | 一次生成多张，挑顺眼的继续改 |
| 担心积分乱扣 | 生成失败会退，记录里能查到结果 |
| 有团队一起用 | 管理后台可以管用户、积分、卡密和历史 |

核心页面是 **「御弟哥哥 · gpt-image-2 生图」**，也就是项目里的 `image-studio.html`。

## 适合谁？

- 小红书、抖音、公众号等内容创作者
- 跨境电商、淘宝、拼多多、独立站卖家
- 需要大量试图的运营和设计同学
- 想搭一个内部 AI 生图工具的小团队
- 想把图片生成、积分、用户、后台放在一起管理的人

说白了：如果你每天都要和“图”“文案”“产品表达”打交道，它就能替你省掉不少重复劳动。

## 画面工坊怎么玩？

打开页面后，基本就是三步：

1. 写清楚你想要的图片

   比如“白底产品图、柔和光线、干净高级、适合亚马逊主图”。

2. 选择尺寸、质量和张数

   尺寸可以选方图、竖图、横图；质量有快速、标准、精细；一次最多生成 5 张。

3. 点生成，然后看记录

   生成完成后，可以预览、下载、放大看，也可以拿同一个提示词重新生图。

如果你有参考图，也可以直接拖进去、粘贴进去，或者点击上传。最多支持 4 张参考图，适合“保留这个产品，但换个背景/光线/构图”这种场景。

## 它有哪些顺手的小功能？

| 功能 | 体验 |
| --- | --- |
| 文生图 | 只写文字也能出图 |
| 图生图 | 上传参考图后继续改 |
| 拖拽上传 | 图片拖到参考图位置就行 |
| 粘贴上传 | 截图或复制图片后直接粘贴 |
| 生成记录 | 生成过的图都能回头找 |
| 提示词重新生图 | 看哪张顺眼，就拿它再试一轮 |
| 图片下载 | 好图直接保存 |
| 积分流水 | 花了多少、退了多少都能查 |
| 管理后台 | 用户、积分、卡密、订单都能管 |

## 关于图片尺寸

页面会尽量按你选的尺寸去请求上游，但图片模型有时会有自己的“小脾气”：你选的是某个尺寸，实际回来的图片可能会略有不同。

目前项目里已经做了适配：

| 页面选择 | 常见实际结果 | 说明 |
| --- | --- | --- |
| 方图 | 接近方图 | 适合头像、封面、产品主图 |
| 竖图 | 竖版图 | 适合小红书、海报、详情图 |
| 横图 | 横版图 | 适合 Banner、场景图、横向展示 |

如果上游返回的尺寸和请求尺寸不完全一样，系统会优先保留原图，不会硬拉伸图片。这样做的好处是画质更稳，不会把图扯变形。

## 关于快速、标准、精细

这三个不是玄学按钮，大概可以这么理解：

| 质量 | 适合场景 |
| --- | --- |
| 快速 | 多试几版、找方向、看构图 |
| 标准 | 日常出图，速度和效果比较均衡 |
| 精细 | 想要更认真、更细致的成图 |

页面会显示大概的上游成本，方便你心里有数。真正给用户扣的是系统里的积分，失败会按规则退回。

## 积分怎么理解？

积分就是系统里的使用额度。

- 生成图片会扣积分
- 生成失败会退积分
- 批量生成时，如果少出图，也会按少的数量退
- 新用户可以配置赠送积分
- 管理员可以在后台充值、发卡密、看流水

不用用户自己算来算去，系统会把记录留好。

## 本地怎么跑？

先安装依赖：

```bash
npm install
```

再启动：

```bash
npm start
```

默认打开：

```text
http://127.0.0.1:3001
```

检查代码有没有明显语法问题：

```bash
npm run check
```

Windows 下也可以用项目里的启动脚本或部署脚本来跑。

## 需要配置什么？

项目根目录里有 `.env.example`，复制一份改名为 `.env`，然后填自己的配置。

最常用的是这些：

| 配置 | 说明 |
| --- | --- |
| `PORT` | 服务端口，默认 3001 |
| `HOST` | 监听地址 |
| `JWT_SECRET` | 登录用的密钥，正式环境一定要改 |
| `ADMIN_USERNAME` | 管理员账号 |
| `ADMIN_PASSWORD` | 管理员密码 |
| `XI_XU_API_BASE_URL` | xixu / new-api 上游地址 |
| `XI_XU_API_KEY` | 上游 API Key |
| `XI_XU_IMAGE_MODEL` | 图片模型，比如 gpt-image-2 |
| `NEW_USER_BONUS_POINTS` | 新用户赠送积分 |

真实密钥、数据库、上传图片都不要提交到 GitHub。

## 管理员能做什么？

管理员后台主要用来管“人”和“钱袋子”：

- 看用户列表
- 给用户加积分
- 看积分流水
- 生成卡密
- 管理生成历史
- 查看订单
- 检查系统配置

如果你是自己一个人用，它就是个总控台；如果你给团队用，它就是小型运营后台。

## 数据会放在哪里？

默认情况下：

| 内容 | 位置 |
| --- | --- |
| 用户和积分 | `data.db` |
| 生成历史 | `data.db` |
| 上传和生成图片 | `uploads/` |
| 本地密钥配置 | `.env` |

这些都属于运行数据，别传到公开仓库里。

建议定期备份：

- `data.db`
- `uploads/`
- 服务器上的 `.env`

## 适合怎么用？

几个比较舒服的用法：

- 先用“快速”多试几张，找到方向后再用“精细”
- 做商品图时先固定产品，再慢慢换背景和灯光
- 遇到好看的竞品图，可以先反推提示词，再改成自己的风格
- 同一个提示词不要只出一张，多出几张通常更容易挑到好图
- 不满意时别急着推倒重写，先改一个变量，比如角度、背景、材质或光线

AI 生图有时候像开盲盒，但盲盒开多了也会摸到规律。

## 当前小提醒

- 上游模型可能会调整尺寸规则，所以实际出图尺寸以生成结果为准
- 图片质量越高，通常成本越高，速度也可能更慢
- 支付功能目前更适合内部测试和人工确认
- `/uploads/` 默认是知道链接就能访问，涉及隐私图时建议额外加保护
- 正式上线前，一定要换掉默认管理员密码和所有示例密钥

## 一句话总结

御弟哥哥 AI 图文创作工作台，就是一个把 **生图、改图、提示词、记录、积分、后台** 放在一起的小工具。

它不追求把页面做得很吓人，也不要求用户懂技术。目标很朴素：让你更快把脑子里的画面变成能用的图片。
