# 产品框架

## 一句话定位

冰侠客是面向小红书创作者和商家运营的 AI 内容工作台，用积分制把图片生成、文案生成、文案改写和图文一体生成串成一个可复用的创作流程。

## 用户和场景

| 用户 | 核心任务 | 当前痛点 | 产品机会 |
|---|---|---|---|
| 个人创作者 | 每周稳定发布笔记 | 选题、标题、配图耗时 | 降低从主题到成稿的时间 |
| 商家运营 | 为商品/门店生成种草内容 | 内容风格不稳定，转化素材缺 | 模板化生成卖点、体验、避坑 |
| 团队管理员 | 控制用量和分发权益 | 账号和积分难管理 | 后台统一看用户、积分、卡密 |

## 北极星指标

**7 日内成功创作并保存的有效内容数。**

有效内容定义：

- 图片：生成成功并进入历史记录。
- 文案：生成或改写成功并进入历史记录。
- 图文：图片或文案至少一项成功，且进入历史记录。

## AARRR 漏斗

| 阶段 | 目标 | 指标 |
|---|---|---|
| Acquisition 获取 | 让用户完成注册 | 访问注册页、注册成功率、邀请码使用率 |
| Activation 激活 | 用户完成第一次创作 | 首次创作成功率、注册到首次创作耗时 |
| Retention 留存 | 用户持续回来创作 | D1/D7 留存、7 日创作次数 |
| Revenue 收入 | 用户购买或兑换积分 | 充值转化率、卡密兑换率、ARPPU |
| Referral 推荐 | 用户传播工具或卡密 | 卡密分发量、邀请注册数 |

## MVP 边界

必须有：

- 登录注册和用户积分。
- 图片、文案、改写、图文一体生成。
- 历史记录和基础统计。
- 管理员充值、卡密、用户管理。
- 内容规范、帮助说明、隐私和条款入口。

暂不做：

- 多团队组织架构。
- 自动发布到小红书。
- 多模型自由选择。
- 复杂素材库和协作审批。

## 体验原则

- 先让用户开始创作，再解释复杂规则。
- 每次消耗积分前后都要明确展示余额变化。
- 失败要给出可行动反馈，失败扣分必须自动退回。
- 历史记录是核心资产，不能只当日志。
- 管理后台以效率为先，少做装饰，多做筛选、批量和导出。

## 缺口清单

| 缺口 | 优先级 | 说明 |
|---|---|---|
| 安全上线基础 | P0 | 静态暴露、支付验签、XSS、密钥轮换 |
| 帮助和规则内容 | P0 | 用户需要知道积分、失败、内容边界 |
| 埋点和漏斗 | P1 | 当前只能看结果，不能分析转化 |
| 模板体系 | P1 | 让高频场景可复用 |
| 生成质量反馈 | P1 | 用户可标记好/差，反哺提示词 |
| 支付正式化 | P1 | 模拟支付不能用于生产 |
| 内容合规审核 | P2 | 降低违规、低俗、侵权内容风险 |
