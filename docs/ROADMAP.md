# 迭代路线

这份路线图按“先稳住能用，再做运营，再做收费上线”的顺序排。

## 当前状态

已经完成或当前可用：

- Node.js + Express + SQLite 单体应用。
- 登录、注册、JWT 鉴权。
- 本地/内网免邀请码注册，公网默认需要邀请码。
- 画面工坊：`gpt-image-2` 文生图、参考图改图、看图写 Prompt。
- 画面工坊尺寸已按当前上游实测适配，保存后记录真实输出：
  - `1254x1254`
  - `1024x1536`
  - `1536x1024`
  - `1672x941`
  - `941x1672`
- 4K 按钮已去掉。
- 尺寸不匹配时保留原图并回传。
- 生成记录支持预览、下载、重试、复用提示词和参考图。
- 小红书图片、文案、改写、图文一体。
- 积分扣费、失败退款、少出图退款。
- 管理后台：用户、积分、历史、卡密、订单、系统状态。
- README 已恢复为详细版本，并保留用户能看懂的大白话说明。

## V0.2 稳定可用

目标：自己用、内部用，不容易出大事故。

- 完成 README 和 `docs/` 同步。
- 保持 5 个当前实测尺寸入口，继续隐藏 4K。
- 记录真实输出尺寸，并在后台能查看尺寸不匹配情况。
- 优化失败错误文案，让用户知道是重试、换尺寸还是联系管理员。
- 注册配置清晰：本地/内网免邀请码，公网需要邀请码。
- 补齐帮助中心里关于积分、失败退款、尺寸变化、参考图改图的说明。
- 确认 `.env`、`data.db`、`uploads/` 不会被部署覆盖。

## 验收口径

- 用户登录后可以完成一次文生图。
- 用户上传参考图后可以完成一次改图。
- 用户上传图片后可以完成一次看图写 Prompt。
- 每次成功创作后，积分、历史、顶部统计同步更新。
- 失败不错误扣分，退款流水可追溯。
- 图片尺寸按真实输出展示。
- 4K 不在页面出现。
- 普通用户不能访问管理员接口。
- 未登录用户不能调用生成接口。
- 生产环境不暴露源码、数据库、日志和测试文件。
